# bx — 基于 brook 的透明全局代理工具（设计文档）

> 日期：2026-06-03
> 作者：gushirui（与 Claude 协作设计）
> 状态：已批准，待实现

## 1. 概述

`bx` 是一个自研的、基于免费 brook 的**透明全局代理**命令行工具（单一 Go 二进制）。
定位：免费版的「类 ipio」——所有系统流量在网络层自动走 brook 加密隧道，中国流量直连，
无需任何程序单独配置代理。完全自有代码、可持续扩展。

**为什么自研**：brook 免费 CLI 只能开 socks5/http 代理；官方透明 TUN 工具 ipio 自 2025 起需
Brook Plus 付费（`--cliToken`），其主体闭源。但 brook 的免费 `connect` 已提供稳定可用的加密
socks5 隧道，我们只需自研「TUN→socks5 + 分流 + DNS + 保护 + 面板」这一层。

## 2. 目标 / 非目标

**目标（MVP）**
- 透明全局：TUN 接管全部 TCP/UDP，对程序零配置
- 中国直连智能分流（域名 + geoip-cn + 用户规则，规则热重载）
- 断线保护：brook 子进程守护+自动重连；kill-switch 防真实 IP 裸奔
- 开机自启（systemd）
- 命令行状态面板（节点/延迟/速率/连接数/代理直连占比）

**非目标（YAGNI，本期不做）**
- 多服务器 / 自动故障转移（先单节点）
- Web 面板（只做 CLI status）
- Windows / macOS（先 Linux）
- 自己实现 brook 协议（brook 当黑盒子进程）

## 3. 整体架构（路线 B：brook 子进程 + 自研外壳）

```
所有流量 ─▶ ① TUN 引擎(netstack) ─▶ ② 分流路由器 ─┬─[直连]─▶ 直接拨号(绕过tun)
                                                  └─[代理]─▶ ③ brook 隧道管理器
                                                              └ brook connect --socks5 127.0.0.1:N ─▶ 服务器 ─▶ 目标
支撑：④ DNS 处理器(fake-IP/防污染)  ⑤ 断线保护+生命周期  ⑥ 状态IPC+CLI
```

**关键边界**：brook 只负责加密隧道（黑盒、稳定）；TUN / 分流 / DNS / 保护 / 面板 / 配置全部为
自有代码。决策（纯逻辑）与 IO（网络）分离，便于纯单测。

## 4. 模块、职责与接口

| 模块 | 职责 | 接口（概念） | 独立测试 |
|---|---|---|---|
| ① TUN 引擎 | 捕获 IP 包，netstack 终结 TCP/UDP，每条新连接交给 Dialer 后双向 splice | `Engine.Start(d Dialer)`；只调 `d.Dial(ctx, meta)` 拿 `net.Conn` | 假 TUN（管道）喂构造包，断言 meta 与字节转发。无真网络 |
| ② 路由器（分流脑） | `{dstIP, port, domain}` → `直连/代理/阻断`。纯函数 | `Router.Decide(meta) Decision` | 表驱动纯单测 |
| ③ Dialer | 问 Router → 直连 `net.Dial`；代理走 brook socks5；阻断报错(kill-switch) | `Dial(ctx, meta) (net.Conn, error)` | mock Router，断言拨号目标 |
| ④ brook 隧道管理器 | 管 `brook connect --socks5 127.0.0.1:N` 子进程：启动/健康检查/重连/延迟 | `Tunnel.SocksAddr() / Healthy() / Stats()` | 注入假命令 stub；健康检查单测 |
| ⑤ DNS 处理器 | TUN 上 53 查询：fake-IP 应答；直连域名国内 DNS、代理域名经 brook 远程解析 | `DNS.Resolve(q) (answer, route)` | mock 上游，断言选路+答案 |
| ⑥ Supervisor | 顶层编排：配置→隧道→TUN→接线→设路由/DNS→信号→kill-switch→退出还原 | `Supervisor.Run(cfg)` | 集成测；kill-switch 策略单测 |
| ⑦ 状态/CLI | 线程安全计数器 + unix socket server；`bx status` 客户端读取打印 | `Stats` + socket + CLI | 计数器单测；渲染用假 payload |
| ⑧ 配置 | 解析+校验 YAML；SIGHUP/文件监听热重载 | `Config.Load(path) / Watch()` | 解析 fixture 断言 |

**接线**：Supervisor 接线 → TUN引擎 → Dialer → Router；DNS 也用 Router；Tunnel 被 Supervisor
拥有、被 Dialer 查；Stats 被各模块写、被 CLI 读。

## 5. 分流 + DNS（Fake-IP 方案）

**难点**：要按域名分流，但 TUN 看到的是 IP 包。解法 = Fake-IP（clash/sing-box 同款）。

**流程**
1. 程序问 DNS `google.com` → DNS 处理器**不真解析**，返回保留段假 IP（`198.18.0.0/15`），记 `fakeIP ↔ 域名`
2. 程序连该假 IP → TUN 捕获 → 路由器反查**确切域名** → 命中"代理" → **把域名交给 brook socks5**（brook 服务器端解析，本地永不解析 → 零 DNS 污染）
3. `baidu.com` 同理 → 反查域名 → "直连" → 用**国内 DNS(223.5.5.5)解析真实 IP** → 直连

**判定优先级**
1. 用户显式规则（域名/网段 强制直连/代理）—— 最高
2. 域名命中 china_domain 列表 → 直连
3. 未命中 → 国内 DNS 解析后做 geoip-cn：中国 IP→直连，否则→代理（代理则丢弃解析结果、仍把域名交给 brook）
4. 无域名的裸 IP 连接 → 对 dstIP 做 geoip-cn

**数据源**：复用已有 `~/.brook/china_domain.txt` + `china_cidr4/6.txt`（geoip 用 CIDR 即可，
不必额外下 MaxMind）；每日 cron 可复用更新。

**UDP/QUIC(HTTP3)**：同样 fake-IP + brook socks5 UDP（brook 原生支持 UDP），不漏走。

## 6. 断线保护 + 生命周期

- **brook 守护**：隧道管理器定期经 socks5 健康检查；挂了/不健康→自动重启 brook，带退避重连。
- **Kill-switch**：隧道不健康期间，Dialer 对"代理"决策连接**返回阻断**（不回退直连），防真实 IP 裸奔；直连流量不受影响。可配 `killswitch: on/off`。
- **退出还原**：捕获 SIGINT/SIGTERM → 撤销路由/DNS、删 TUN、停 brook；systemd 停服务同理。

## 7. 配置 + CLI

**配置 `~/.config/bx/config.yaml`**
```yaml
server: "brook://..."          # 或 server: host:port / password: xxx
killswitch: true
dns:
  china: 223.5.5.5
  fakeip_cidr: 198.18.0.0/15
rules:                          # 热重载，优先级从上到下
  - direct: ["*.internal.com", "10.0.0.0/8"]
  - proxy:  ["*.openai.com"]
lists:
  china_domain: ~/.brook/china_domain.txt
  china_cidr:   ~/.brook/china_cidr4.txt
```

**CLI**
- `bx up` / `bx down`：启停
- `bx status`：节点、隧道延迟、上下行速率、活跃连接、代理/直连占比
- `bx reload`：热重载规则（或 SIGHUP）
- `bx install`：生成并启用 systemd 服务（开机自启）

## 8. 测试策略

- **分流脑（Router/DNS 决策）**：表驱动纯单测，无 root 无真网络 —— 覆盖最关键逻辑
- **Dialer**：mock Router，断言拨号目标
- **TUN 引擎 / 隧道管理器**：集成测（本地假 socks5 + 假 TUN/管道）
- **配置**：fixture 解析 + 校验断言

## 9. 依赖与构建

- 单一静态 Go 二进制（CGO_ENABLED=0）
- gVisor netstack（TUN 用户态协议栈）
- 一个 socks5 client 库（连 brook 本地 socks5）
- 运行时依赖 brook 二进制（子进程；已在 `~/.nami/bin/brook`）
- GOPROXY 设 goproxy.cn（国内拉包可靠）
- 模块路径：`github.com/getbx/bx`

## 10. 里程碑（每步可独立验证）

1. 骨架：go.mod + CLI 框架（cobra/urfave）+ 配置解析 + 单测脚手架
2. **分流脑**：Router + 规则引擎 + geoip(CIDR) + fake-IP 映射 + 全套单测 ← 先啃硬骨头
3. brook 隧道管理器：子进程 + 健康检查 + 自动重连

5. DNS 处理器：fake-IP + 防污染 + 中国直连落地
6. 断线保护 + kill-switch
7. status 面板 + systemd 自启 + `bx install`
